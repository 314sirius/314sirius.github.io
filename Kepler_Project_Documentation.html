Kepler Exoplanet Candidate Filtering

Dataset & Feature Engineering

Source: NASA Exoplanet Archive - Kepler Cumulative KOI Table

The dataset tracks 9,564 observation entries across 49 astronomical columns. The engine parses features split into distinct functional groups:

  • Transit (Planetary) Variables: koi_period, koi_duration, koi_depth, koi_prad, koi_insol.
  • Stellar Characteristics: koi_steff, koi_srad, koi_slogg.
  • Target Label Configuration: Binomial target filtering to focus on CONFIRMED planets vs FALSE POSITIVE.

🛠️ Tech Stack & Methodology

Data Processing: Pandas, NumPy, MinMaxScaler

Analytical Formulations: SymPy

Machine Learning: L1-Regularized Logistic Regression, Decision Tree, Random Forest Ensemble.

Validation: ROC-AUC, Precision, Recall, Confusion Matrices, PDP.

🔑 Key Analytics Insights

  • Non-Linear Reality: Tree-based models outperformed linear ones due to threshold-based physical constraints.
  • Gateway Variable: log_koi_prad < 0.35 is the critical threshold for validation.
  • Orbital Window: Random Forest isolated a specific window (0.3 to 0.7 log scale) as physically relevant.
